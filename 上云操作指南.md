## 四、部署流程回顾

### 4.1 初始化 Git 仓库

```bash
cd e:/Code/idle-rpg
git init
git config user.name "li_ts"
git config user.email "li_ts@idle-rpg.local"
git add -A
git commit -m "feat: Idle Chronicles 挂机RPG游戏 — 初版完整代码"
```

### 4.2 配置 .gitignore

```
node_modules/       # 依赖不上传（服务器npm install安装）
client/dist/        # 构建产物不上传（服务器构建）
server/storage/     # 用户数据不上传（含密码哈希）
.env                # 敏感配置不上传
```

### 4.3 配置 JWT 密钥持久化

**`ecosystem.config.cjs`** — PM2 启动时注入环境变量：

```js
module.exports = {
  apps: [{
    name: 'idle-chronicles',
    script: 'server/src/index.js',
    env: {
      PORT: 3001,
      JWT_SECRET: '固定随机密钥'  // 重启不掉Token
    }
  }]
};
```

**`server/src/auth.js`** — 无环境变量时直接报错退出：

```js
const JWT_SECRET = process.env.JWT_SECRET || (() => {
  console.error('FATAL: JWT_SECRET 环境变量未设置！');
  process.exit(1);
})();
```

### 4.4 推送代码到 GitHub

```bash
# 生成本机 SSH 密钥
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""

# 显示公钥 → 添加到 https://github.com/settings/keys
cat ~/.ssh/id_ed25519.pub

# 设置远程地址为 SSH 并推送
git remote add origin git@github.com:Li-Hee/idle-rpg.git
git push -u origin master
```

> **为什么用 SSH？** 国内网络 HTTPS 连 GitHub 经常超时，SSH 端口稳定。

### 4.5 VPS 首次部署

```bash
# 1. 安装 Node.js 20 (CentOS 系用 dnf)
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo dnf install -y nodejs git

# 2. 克隆代码
cd ~
git clone https://github.com/Li-Hee/idle-rpg.git
cd idle-rpg

# 3. 安装依赖并构建前端
cd server && npm install && cd ..
cd client && npm install && npx vite build && cd ..

# 4. 创建数据目录
mkdir -p server/storage

# 5. 安装 PM2 并启动
sudo npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

> **注意：** Alibaba Cloud Linux 是 CentOS 系，用 `dnf` 而非 `apt-get`。

### 4.6 VPS 配置 SSH 到 GitHub

```bash
# 在 VPS 上生成密钥
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""

# 验证 → 出现 "successfully authenticated" 即成功
ssh -T git@github.com

# 切换远程地址为 SSH
cd ~/idle-rpg
git remote set-url origin git@github.com:Li-Hee/idle-rpg.git
```

## 五、Webhook 自动部署

### 5.1 工作原理

```
你 git push → GitHub 发 POST 到 VPS → Express 收到请求
→ exec("git pull + npm install + vite build + pm2 restart")
→ 输出写入 deploy.log
```

### 5.2 Webhook 端点代码

**`server/src/index.js`** 中的关键路由（在 JWT 中间件之前）：

```js
app.post('/api/webhook/deploy', (req, res) => {
  res.json({ ok: true, msg: 'deploy triggered' });
  const cmd = 'cd /home/admin/idle-rpg && git pull origin master '
    + '&& cd server && npm install '
    + '&& cd ../client && npm install && npx vite build '
    + '&& pm2 restart idle-chronicles';
  exec(cmd, (err, stdout, stderr) => {
    const log = `[deploy ${new Date().toISOString()}] ...`;
    require('fs').appendFileSync('/home/admin/idle-rpg/deploy.log', log);
  });
});
```

### 5.3 GitHub Webhook 配置

- **地址：** http://47.95.252.163:3001/api/webhook/deploy
- **事件：** Just the push event
- **内容类型：** application/json

每次 `git push` 后约 15-30 秒线上自动生效。

## 七、用户名保护

### 7.1 黑名单

`server/src/auth.js` 中注册时检查，以下用户名不可注册：

**英文：** admin, root, system, gm, mod, owner, master, server, staff, test, guest, anonymous, idle, chronicles 等

**中文：** 管理员, 系统, 客服, 官方, 管理, 超级管理员, 版主

### 7.2 代码

```js
const reserved = new Set([
  'admin','administrator','root','system','sys','gm','game','server',
  'staff','mod','moderator','owner','master','webmaster','host',
  '管理员','系统','客服','官方','管理','超级管理员','版主',
  'admin1','admin2','root1','test','guest','anonymous',
  'idle','chronicles','idlechronicles','idle-chronicles'
]);
if (reserved.has(username.toLowerCase())) {
  return res.status(403).json({ error: '该用户名已被保留，请换一个' });
}
```
